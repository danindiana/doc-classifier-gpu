## Bugs fixed this session

### Bug 1 — BrokenPipeError cascade from `imap_unordered` + worker crash

**Symptom:** Hundreds of interleaved `BrokenPipeError: [Errno 32] Broken pipe`
tracebacks from ForkPoolWorkers (e.g. 518, 522...) when sort_docs.py processed
PDFs. Each crash flooded stderr and terminated all remaining workers.

**Root cause:** `pool.imap_unordered()` raises `WorkerLostError` when any worker
crashes (fitz SIGSEGV on corrupt PDF). The `except Exception` block catches it and
falls out of the `with Pool` context. `pool.__exit__()` unconditionally calls
`pool.terminate()`, closing all IPC pipes. Every in-flight worker trying to send
its result gets `BrokenPipeError`. This is not a recoverable state — the entire
pool is dead and remaining files are abandoned.

**Fix (`sort_docs.py`, commit `9ed00e8`):**
Wrapped the `with Pool` block in a `while files_todo` restart loop:
- `done_paths: set` — tracks path strings that successfully returned results
- `fail_counts: Counter` — how many times each path has appeared in a crash
- `MAX_FAILS = 2` — files that crash twice are blacklisted to `no_text_files`
- `progressed == 0` guard — bail if an entire pool run produces no results

After each pool exit (including crash-triggered ones), the loop restarts with
only the unfinished files. One worker death no longer aborts the whole run.

**Lesson:** → LESSONS_LEARNED.md #11

### Bug 3 — bge-m3 `encode-batch=512` OOM crashed embed thread

**Symptom:**
```
torch.OutOfMemoryError: CUDA out of memory. Tried to allocate 2.94 GiB.
GPU 0 has a total capacity of 15.47 GiB of which 1.24 GiB is free.
```
Thread `Thread-3 (_embedding_worker)` crashed. Extraction pool kept running,
filling `embed_q` with no consumer. Run stalled.

**Root cause:** bge-m3 weights + overhead occupy ~14.2 GB on the RTX 5080 (16 GB).
Default `--encode-batch 512` attempts a single forward pass on 512 text chunks
(~512 × 1000 tokens). XLM-Roberta activation memory for this batch requires
2.94 GB — exceeds the 1.24 GB headroom.

**Fix — two-part (commit `9776e6b`):**

1. **OOM recovery loop in `embed_batch()` (`sort_docs.py`):**
   On `OutOfMemoryError`, clears CUDA cache, halves `encode_batch` (min 4),
   retries. Self-heals without crashing the embed thread.

2. **Relaunched with `--encode-batch 32`** — fits comfortably in ~1.2 GB
   headroom. 32 chunks × ~1000 tokens → manageable activation memory.

**Lesson:** → LESSONS_LEARNED.md #13
